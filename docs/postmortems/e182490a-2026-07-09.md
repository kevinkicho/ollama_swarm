# Postmortem: run `e182490a` (council preset, ~21.5 min crash)

**Preset:** council · **Duration:** ~21.5 min wall-clock · **Stop:** crash  
**Run ID:** `e182490a-907b-439b-b0a5-2c7df9859734`  
**Workspace:** `superconducters_07062026` (external clone)

## Symptom

Run applied 4 commits during execution, then crashed with:

```
stopReason: crash
stopDetail: Cannot complete todo t5: status=failed
```

Summary reported `filesChanged: 0` despite real git commits. ~2.6 minutes of orphan provider activity continued after agents were marked stopped.

## Root cause chain

1. **Todo t5 hit the 20-turn tool cap** during literature/research work — worker could not finish in time.
2. **Council reaper (10 min TTL)** marked the in-progress todo `failed` while the worker was still running.
3. **Worker called `TodoQueue.complete()`** on a successful result after the reaper had already failed the todo.
4. **`complete()` threw** (`status=failed`) — uncaught, propagated as run crash.
5. **No crash abort** — in-flight provider calls were not cancelled, causing post-stop orphan activity (~2.6 min).
6. **`filesChanged` summary** used the wrong git baseline (post-run porcelain vs run-start baseline).

Secondary contributors (performance / durability, not primary crash):

- Bash tool retry storm on repeated command failures (no backoff).
- Runtime-fix todo dedup missing — duplicate repair todos queued.
- Large hunk replacements (up to ~100KB) increased latency and failure risk.
- Per-agent contract explore duplicated repo tours (N× explore cost).

## Fixes applied (2026-07-09)

| Issue | Fix |
|-------|-----|
| Reaper race | `TodoQueue.complete()` accepts late success when failure reason is reaper timeout; council worker uses safe complete path |
| Crash abort | `beginRunShutdown()` on crash via `runFinallyHooks` |
| Tool-turn caps | Literature/research caps raised (40/35/40) in `shared/toolProfiles` + council worker |
| Bash storms | Error backoff after 4 consecutive failures in `ToolDispatcher` |
| Runtime-fix dedup | Guard in `councilDecisions.ts` |
| `filesChanged` | Git baseline captured at run start in `DiscussionRunnerBase` / `CouncilRunner` |
| Reaper TTL | Council worker reaper 10 → 15 min |
| Hunk size | 32KB soft cap in worker prompts + council/blackboard runners |
| Shared explore | `runCouncilSharedExplore` wired in `councilAdapter.runContractDerivation` (was blackboard-only) |
| Mid-run limits | `POST /api/swarm/reconfig` — extend rounds, wall-clock cap, token budget; Brain `RECONFIG:` + Apply button |

## Not the cause

- User stop (`stopReason` is `crash`, not `user`)
- Provider transport failure at start (run progressed through multiple execution cycles)
- Missing council contract (contract + execution phases ran)
