## Root cause chain

### 1. No exit contract

Transcript opens with contract failure pattern from prior investigation:

> `Council contract: 0 drafts survived parsing — proceeding without contract.`

All agents returned mission-only JSON, not full contract with `criteria[]`. Without contract:

- Standup logs `0 criteria, 0 unmet` every cycle
- Audit / criteria promotion loop does not constrain or stop the run
- `rounds: 0` autonomous mode → infinite standup cycles

### 2. Standup synthesis parse failures → empty queue

When agents produce standup JSON arrays, agent-1 `synthesizeStandup()` must parse a **todo** array. Failures or empty merges yield:

> `[Standup] Synthesized 0 proposals into unified plan.`

**13 cycles** hit this. No todos → execution drain logs `0 done, 0 failed, 0 skipped` and the council immediately starts the next standup.

### 3. Execution failures when todos *did* enqueue

When 5–6 todos were enqueued (cycles 3–4, 10–12, 15–16, 21–24, 26), workers failed heavily:

| Failure category | Approx. count (from transcript) |
|------------------|--------------------------------:|
| JSON parse failed (worker response) | 22 |
| Primary failed (`hunks` schema / missing fields) | 17 |
| Intra-stream loop | 6 |
| Other / repair exhaustion | 26 |

Common patterns:

- Model returns **markdown or prose** instead of worker JSON (`###`, `def predict`, Python snippets)
- Model returns **invalid hunks** (`hunks.0.op` wrong discriminator, `hunks: Required`)
- Model emits **streaming loops** (repeated suffix / identical deltas) → aborted
- **Duplicate-test todo** repeatedly assigned; workers skip or fail on `tests/tests_predict_tc.py`
- Workers **skip** when they believe work is already done (database, predict_tc, online_research_summary) — good behavior, but council keeps re-proposing same todos

### 4. Same todos recycled

Standup + synthesis keep rediscovering the same gaps (stub `predict_tc.py`, empty DB, duplicate test file, pipeline script, research summary). Even successful commits (e.g. `f4a3e35`, `9f7a022`) did not stop the loop because **no contract criteria** marked progress and synthesis does not read git state holistically.

### 5. Late-run transport stress

Debug log tail shows `agent-2` **retrying** (`Ollama idle timeout: no body data for 180000ms`) immediately before user stop — long prompts + cloud provider stalls compound failure rate.

## Why “so many fail per cycle”

Not random flake — **structural**:

1. **Queue fills with 6 parallel todos** each execution cycle
2. **deepseek-v4-flash** often violates strict JSON/hunk envelope under tool-free worker prompts
3. **Repair + failover** paths still fail when model returns non-JSON
4. **Failed todos retry** within the same cycle (drain loop), inflating per-cycle fail counts in the Drafts tab
5. **Skips count as non-success** but are often correct (“file already complete”) — council treats them as unfinished work and re-queues similar todos next standup
