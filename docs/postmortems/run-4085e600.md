# Postmortem: run `4085e600` (12s crash before contract)

## Symptom

Run completed goal-analysis + web-research pre-passes, then crashed before first-pass contract. UI showed agents going to `stopped` ~12s after start. No board todos, no contract, 0 planner turns recorded.

## Root cause

```
stopReason: crash
stopDetail: Cannot read properties of undefined (reading 'markStatus')
```

`runFirstPassContractOrchestrator` → `tryCouncilContract` calls `emitAgentActivity(agent, ctx.manager, …)`, but `contractContext()` did **not** wire `manager` onto `ContractContext`. Same gap on `plannerContext()` for `runPlanner` recovery activity labels.

Pre-passes used `ctx.getManager()` from lifecycle context (wired correctly). Contract/planner paths used extracted contexts missing `manager` — crash on first `emitAgentActivity` after research pre-pass.

## Fix (2026-07-08)

- `contextBuilders.ts`: add `manager: r.opts.manager` to `contractContext` and `plannerContext`
- `contractBuilder.ts` / `plannerRunner.ts`: declare `manager: AgentManager` on context interfaces
- `contextBuilders.test.ts`: regression test for manager wiring

## Not the cause

- User stop (summary shows `stopReason: crash`, not `user`)
- Research brief rejection (notes were captured: 1623 chars)
- Yahoo/finance-specific logic
- Streaming UI issues
